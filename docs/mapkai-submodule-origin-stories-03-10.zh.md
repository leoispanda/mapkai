学科起源
030 社会科学、新闻与信息（未进一步定义）

一句简介
个人声音堆到一起时，社会问题先需要一个总览入口。

起源故事
一座城市的档案室里，桌上摊着三类材料：税册、报纸剪报和居民来信。过去，人们容易把这些东西分开处理。税册属于官员，报纸属于编辑，来信属于个人遭遇。谁失业、谁搬家、谁抱怨物价，常被看成各自生活中的事。

可是材料一排开，个人事件开始露出共同形状。几条街的租金同时上涨，报纸连续把同一类事故放到角落，来信里反复出现同一种通勤困难。若只问某个人为什么失败，很多解释说得过去；若把记录按时间、地点和人群排开，问题不再只属于任何一个人。

这个总览入口因此出现：先让学习者进入人与行为、公共信息、制度、报道和社会证据的共同地带，再决定它应细分到社会科学、新闻或信息研究。它不是传统单科，而是社会问题进入知识地图的前厅。

为什么这个学科需要出现
社会、媒体和信息问题常常交织在一起。没有总览入口，学习者会过早把问题塞进心理、政治、新闻或档案等细格，忽略个人经验怎样被制度、传播和公共记录连接起来。

核心转向
从「把事件看成个人或单一机构问题」，转向「先识别它是否需要社会、媒体和信息的共同解释框架」。

关键人物 / 事件 / 工具
人口普查、社会调查、报纸编辑台、公共档案、民意记录和城市统计表，是这一入口的重要工具。它们让分散经验可以被比较、公开和讨论。

后来长出的分支
它连接到社会与行为科学、新闻与信息、公共领域研究、社会调查、媒介研究和跨学科公共问题项目。

反思问题 / 小测
当许多人都说「这是我的问题」时，你会不会先问它是否已经成为一个社会问题？

学科起源
031 社会与行为科学

一句简介
社会与行为科学出现，是因为人的选择总被环境和他人改写。

起源故事
早期城市调查员拿着本子走进街区，本来只想记录家庭收入、职业和住房情况。过去，解释人的行为常靠性格、道德或个人意志。谁贫穷，可能被说成不够勤奋；谁犯罪，可能被说成天性不好；谁换工作，可能被说成选择不同。

可街区记录让这种解释变得不够。几个家庭收入不同，却都被同一条工厂停工影响；几个年轻人性格各异，却在同一条拥挤街道上被同一种机会和风险包围。调查员开始把个体选择放回家庭、群体、价格、制度、规范和情绪之中。

问题随之改变：不只是「这个人为什么这样做」，还要问「哪些条件让这种行为更可能出现」。当行为需要通过比较、调查、实验、访谈和统计来理解，社会与行为科学就有了自己的对象和方法。

为什么这个学科需要出现
人的行为既不是纯粹个人意愿，也不是简单环境决定。现代社会需要解释选择、规范、身份、经济激励、心理过程和群体关系怎样共同塑造行动。

核心转向
从「用个体品格解释行为」，转向「研究个体、群体、制度和环境如何共同影响行为」。

关键人物 / 事件 / 工具
城市社会调查、心理实验、经济模型、民意调查、民族志、统计分析和政策评估，是这一学科成形的重要工具。

后来长出的分支
后来发展出经济学、政治科学、心理学、社会学、文化研究、行为研究和公共政策分析等方向。

反思问题 / 小测
当你解释一个人的选择时，有没有同时看见他所在的街区、规则和人群？

学科起源
032 新闻与信息

一句简介
消息太多以后，人们需要知道什么能进入公共视野。

起源故事
一间报馆的编辑桌上，电报、来信、讣告、法院记录和市场价格挤在一起。过去，消息常靠口耳相传、公告栏或少数官方通知流动。人们关心发生了什么，却不总能知道消息从哪里来、是否可靠、为什么被放在显眼位置。

城市、印刷、通信和公共政治让消息数量暴涨。编辑必须决定哪条放头版，图书馆员必须决定怎样编目，档案员必须判断哪份记录值得长期保存。一个事实不只是被发现，还要被采集、核实、排列、保存和传播。

新闻与信息的入口就在这些工作台上形成。它不只关心内容本身，也关心信息如何被选择、组织、检索、解释和公开。当社会依赖公共事实作判断时，信息的流通方式就成了学科问题。

为什么这个学科需要出现
现代社会需要可靠报道、可追溯记录、可检索知识和公共信息秩序。没有这一学科，消息容易变成噪音，档案容易沉没，公众也难以判断哪些事实被看见、哪些被遗漏。

核心转向
从「把消息当作自然流动的内容」，转向「研究信息如何被采集、验证、组织、保存和传播」。

关键人物 / 事件 / 工具
报馆、通讯社、图书馆目录、档案制度、新闻伦理、信息检索系统和数字平台，是这一 subject 的关键支撑。

后来长出的分支
后来发展出新闻报道、编辑、信息科学、图书馆学、档案学、媒介研究和数据新闻。

反思问题 / 小测
当一条消息出现在最显眼的位置时，你会问它是否真实，也会问它为什么在那里吗？

学科起源
038 社会科学、新闻与信息跨学科项目与资格

一句简介
公共问题常要调查表、采访和档案一起工作。

起源故事
一个社区要解释夜间噪音投诉。桌上有报警记录、居民访谈、街区地图、社交媒体帖子和记者拍回来的照片。过去，社会科学可以做调查，新闻可以写报道，档案可以保存记录；每种方法都有自己的桌子和标准。

但这个问题不听学科边界。报警记录显示投诉集中在三条街，访谈说老人和夜班工人的感受完全不同，照片显示施工围挡改变了人流，社交媒体又把个别事件放大成城市恐慌。单独使用一种材料，会让问题变窄。

跨学科项目因此把多种证据放到同一张桌上：数字指出哪里异常，采访解释异常为何被感受，报道让公共讨论发生，档案保证后来还能追溯。它不是拼盘，而是围绕公共问题组织方法。

为什么这个学科需要出现
现实公共问题常同时涉及行为、制度、媒体呈现和信息证据。没有跨学科入口，学习者容易只会调查、只会报道或只会整理资料，却不会让不同证据互相校正。

核心转向
从「分别掌握社会调查、新闻和信息技能」，转向「围绕同一公共问题组织多种证据和表达」。

关键人物 / 事件 / 工具
社区研究、调查报道、公共数据、档案项目、参与式地图和混合方法课程，是这一 subject 的重要工具。

后来长出的分支
它连接到数据新闻、公共问题项目、政策传播、社区档案、数字社会研究和 civic tech。

反思问题 / 小测
当一张表和三段采访给出不同答案时，你会让谁修正谁？

学科起源
039 社会科学、新闻与信息（未另分类）

一句简介
有些公共问题先暴露边界，再决定归属。

起源故事
研究中心收到几份新项目：网络迷因如何影响投票情绪、城市传言怎样改变抢购、平台评论如何重塑商店声誉、匿名社区如何保存灾害记忆。过去，工作人员会先找熟悉格子：像政治就放政治科学，像新闻就放传播，像档案就放信息研究。

可这些项目常常不肯安静落座。网络迷因既是文化表达，也是政治传播；平台评论既像市场信息，也像公共评价；灾害记忆既有档案，又有社群自救。若硬塞进旧格子，问题会被切掉一半。

NEC 入口的作用不是收纳混乱，而是给边界案例一个有纪律的停靠点。它要求研究者写清为什么现有分类不合适、哪些 related paths 相关、何时应复查并分流。

为什么这个学科需要出现
新媒介、新社会行为和新信息形态不断出现。没有边界入口，分类系统会过早扩张或错误归类，影响内容治理和学习路径。

核心转向
从「遇到新现象就硬匹配旧标签」，转向「先记录边界理由，再用证据决定归属、项目包或新路径」。

关键人物 / 事件 / 工具
研究审查表、分类复查、平台数据、社区档案和跨领域评议，是这一入口的核心工具。

后来长出的分支
部分内容会进入社会行为、新闻传播或信息研究；部分更适合作为 project pack 或 related path，而不是新 submodule。

反思问题 / 小测
一个新现象放不进旧盒子时，你会急着造盒子，还是先写清它卡在哪里？

学科起源
040 商业、管理与法律（未进一步定义）

一句简介
交易、组织和规则交在一起时，先需要总览入口。

起源故事
一份创业培训手册摆在桌上，里面同时写着账本、客户、合同、税务、团队分工和风险责任。过去，这些内容可以分开学：会计管记录，营销管顾客，管理管组织，法律管规则。边界清楚时，分开很有效。

可是现实项目常从一张订单开始就把它们绑在一起。价格写错会影响利润，合同条款会改变交付责任，客户承诺会影响现金流，团队流程又决定问题能不能被发现。若学习者还没进入具体分支，先需要知道商业、行政和法律共同处理什么。

这个总览入口因此出现。它不是替代会计、金融、管理、营销或法律，而是帮助学习者先看见组织如何用记录、交易、规则和责任来维持可信行动。

为什么这个学科需要出现
商业活动不是单纯买卖，也不是单纯管理。它牵涉资源配置、组织分工、财务记录、市场信任和法律责任。总览入口能防止学习者只看一个环节。

核心转向
从「把钱、客户、组织和规则分开学习」，转向「先理解交易与组织如何在责任框架中运作」。

关键人物 / 事件 / 工具
账簿、合同、公司制度、税务规则、市场交易、办公室流程和商业学校课程，是这一入口的重要支撑。

后来长出的分支
它连接到商业行政、会计税务、金融保险、管理、营销、销售、法律、合规和组织治理。

反思问题 / 小测
当一笔交易失败时，你会只看价格，还是同时看记录、承诺和规则？

学科起源
041 商业与行政

一句简介
商业与行政出现，是因为组织需要把资源、任务和责任排成可运行的流程。

起源故事
一家早期贸易行的仓库里，货物、账本和信件堆在一起。老板最初靠熟人、记忆和经验安排买卖：谁去码头验货，谁向客户收款，谁把银钱记入账本。规模小时，这种办法灵活，也省去许多手续。

订单增多后，错误开始反复出现。货物到了，账本没改；客户已付款，仓库还在催；一个伙计知道折扣，另一个却按原价开票。问题不再只是某个人粗心，而是任务、记录和责任没有稳定连接。

商业与行政的对象就在这些反复错位中形成：怎样设计流程、记录资金、安排人员、服务客户、监督风险，让组织不靠某一个人的记忆运转。管理和行政不是文书装饰，而是把行动变成可交接、可追踪、可改进的系统。

为什么这个学科需要出现
现代组织要处理订单、客户、资金、员工、供应商和规则。没有商业与行政知识，组织会依赖临时经验，规模一大就难以控制成本、质量和责任。

核心转向
从「靠个人经验做买卖」，转向「用流程、记录、分工和管理机制组织商业行动」。

关键人物 / 事件 / 工具
复式记账、公司制度、办公室管理、销售体系、市场营销、管理理论和信息系统，是这一学科的重要工具。

后来长出的分支
后来发展出会计、金融、管理、营销、办公室工作、销售、人力资源、运营和创业管理。

反思问题 / 小测
如果一项工作只有某个人记得，组织真的掌握了它吗？

学科起源
042 法律

一句简介
法律成为学科，是因为规则必须能超出个人关系稳定适用。

起源故事
一间地方审理厅里，桌上放着两份相似纠纷：同样是借款不还，一份牵涉亲族，一份牵涉陌生商人。过去，很多争端靠长者、习俗、权威或人情调停。熟人社会中，这些方式能迅速止争，也能保留关系。

可是城市交易、财产流动和政治共同体扩大后，人情不再够用。陌生人之间需要知道同类案件会怎样处理；弱势一方需要不靠关系也能被听见；统治者的命令也需要被公开规则限制。

法律的对象因此变得清楚：规则如何制定、解释、适用和执行；程序如何保护判断；权利、义务和责任如何在不同案件中保持可说明的一致性。

为什么这个学科需要出现
没有法律，争端容易靠身份、权力或临时情绪解决。现代社会需要公开规则、程序保障、可预测责任和对权力的约束。

核心转向
从「靠习俗、人情或权威处理争端」，转向「用公开规则、程序和理由处理权利义务」。

关键人物 / 事件 / 工具
成文法、判例、法院、契约、权利理论、正当程序和法律教育，是法律学科成形的重要支撑。

后来长出的分支
后来发展出民法、刑法、行政法、商法、国际法、宪法、程序法、合规和法律伦理。

反思问题 / 小测
当熟人和陌生人遇到同类争端时，规则能不能同样站得住？

学科起源
048 商业、行政与法律跨学科项目与资格

一句简介
组织问题常同时要求商业判断和法律边界。

起源故事
一家食品公司准备上线新配送服务。桌上有市场计划、供应合同、员工排班、保险条款和监管要求。过去，营销团队关心增长，运营团队关心交付，法务关心责任。每一边都说得对，却常在最后一刻互相卡住。

项目真正运行前，问题变成一张共同表：广告承诺是否超过实际能力，配送员安全谁负责，客户数据如何保存，供应商违约怎样处理。只懂商业会低估规则风险，只懂法律又可能看不见业务流程如何产生风险。

跨学科项目的价值在于把规则嵌入组织行动，而不是事后补文件。它让商业、管理和法律在同一个流程图里看见彼此的约束。

为什么这个学科需要出现
合规、治理、平台业务、金融服务和组织风险都不能只由单一学科处理。现实问题要求商业模式、流程设计、责任分配和法律规范同时进入判断。

核心转向
从「商业先做、法律后审」，转向「在设计组织行动时同步安排规则、责任和监督」。

关键人物 / 事件 / 工具
合规体系、风险管理、公司治理、监管报告、合同流程和跨部门项目，是这一 subject 的关键工具。

后来长出的分支
它连接到合规、公司治理、商业法、监管科技、风险治理、合同管理和伦理商业。

反思问题 / 小测
如果规则只在最后盖章，它真的参与了设计吗？

学科起源
049 商业、管理与法律（未另分类）

一句简介
边界交易需要被看见，而不是被匆忙塞进旧格子。

起源故事
分类员收到几项新业务说明：平台代售虚拟物品、会员积分交换服务、共享空间里的短期办公权益、自动订阅里的法律告知。过去，商业内容可按销售、金融、管理或法律分开。可这些新形式常同时像货物、服务、金融权益和合同安排。

如果硬把它们放进单一分类，责任会跟着偏移：算货物就看质量，算服务就看体验，算法律问题就看条款，算金融就看风险。每个标签都能解释一部分，也会遮住一部分。

NEC 入口让这些边界业务先被记录，并要求说明为什么现有分支不够、哪些规则相关、何时需要复查。它不是商业创新的垃圾桶，而是防止分类过早下结论的缓冲区。

为什么这个学科需要出现
商业形态和监管对象持续变化。没有边界入口，分类系统容易把新交易误放到旧责任框架里，导致学习者和维护者误判风险。

核心转向
从「按最相似标签立即归类」，转向「先识别交易结构、责任边界和复查条件」。

关键人物 / 事件 / 工具

起源故事
服务目录里出现新项目：共享厨房体验、线上陪伴、宠物友好旅行协调、灾后临时接待、平台化上门服务。过去，分类员会把它们放进餐饮、旅游、个人服务或安保。

可这些服务往往同时跨过几个窗口。线上陪伴既像个人服务，也牵涉福利和信任；共享厨房既像餐饮，也像社区活动；灾后接待既是服务，也是安全和卫生安排。单一标签会遮住关键责任。

NEC 入口让这些新服务先被记录，标明 related paths、风险和复查条件。它不是杂项仓库，而是服务创新进入 taxonomy 前的缓冲区。

为什么这个学科需要出现
服务业不断出现混合体验、平台服务和临时服务。没有边界入口，分类系统容易误导内容生产和责任判断。

核心转向
从「按最像的服务窗口归类」，转向「先识别体验结构、责任边界和复查路径」。

关键人物 / 事件 / 工具
服务创新、平台运营、服务恢复、信任机制、体验评估和分类复查，是这一入口的重要工具。

后来长出的分支
部分内容会进入个人服务、运输、卫生或安全；部分更适合作为 project pack、案例库或 related path。

反思问题 / 小测
