## 3. Product Memory

MapKAI means:

```text
Map Knowledge with AI
```

Product concept:

MapKAI is a modular knowledge map and learning path system. It helps learners turn broad, scattered knowledge into clear maps, structured categories, guided learning paths, and eventually quiz-based validation.

Core metaphor:

```text
Unknown ocean -> snow mountain -> land -> green land
```

Meaning:

- Ocean = knowledge not explored yet.
- Snow mountain = first understanding.
- Land = knowledge understood well enough to explain.
- Green land = stronger mastery.

Main public modules:

```text
/           Home
/map        Knowledge map + blind quiz flow
/categories Knowledge classification system
/categories/:code Category detail pages
/learning   Learning path module
```

Module relationship:

- Map shows the big-picture learning landscape and quiz progress.
- Categories stores the official knowledge structure.
- Learning turns categories/fields into guided paths.
- Question banks should be internally tied to category codes, but normal users should not see those codes during quiz flow.

Founder mode:

Founder mode is for internal management only. It can show category codes, subject source, and maintenance details. Normal users should not see internal codes, subject IDs, database keys, or development notes.

## 4. Design Memory

Preferred visual direction:

- Clean
- Warm
- Premium
- Calm
- Product-like
- Slightly playful, but not childish
- Atlas / knowledge map feeling
- Educational, but not school-like
- Clear first-time visitor understanding

Preferred language style:

- Simple
- Direct
- Human
- Not overly abstract
- Not too internal-development oriented
- Metaphor is allowed only when it supports clarity

Avoid:

- Childish cartoon style
- Dark game UI
- Overly busy fantasy maps
- Too many labels on the map
- Internal wording such as "MVP", "do-not-touch boundaries", "semi-manual", "prove the template" in public-facing copy
- Showing category codes to normal users
- Making one category, especially 04, feel more important than all others on public pages

Map image direction:

The future map should ideally be designed as layered assets:

```text
Base ocean layer
Separate island/continent layers
Optional snow/land/green state overlays
Optional glow/progress overlay layers
Final preview composition
```

Reason:

Layered assets allow the site to light up specific land areas after users answer questions, without regenerating the full image every time.
