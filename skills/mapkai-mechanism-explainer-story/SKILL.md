# MapKAI Mechanism Explainer Story Writer

Use this skill for mathematical, statistical, computational, algorithmic, economic, or formula-based knowledge.

## Purpose

Explain how a concept works. This mode should prioritize mechanism, accuracy, and checkable examples over mystery or literary beauty.

## Best For

- Mathematics
- Statistics
- Computer science
- Algorithms
- Economics
- Finance
- Data science
- Logic
- Systems modeling
- Formula-based ideas

## Core Structure

1. Why this concept is needed
2. A concrete problem
3. The old intuitive approach
4. Why the old approach is insufficient
5. Step-by-step mechanism
6. A small example
7. Concept name
8. Boundary or common mistake
9. Reflection question

## Hard Rules

- Do not hide the mechanism behind a vague story.
- Do not use metaphor if it damages accuracy.
- If a formula is needed, include a minimal formula.
- If a calculation is needed, show a small checkable example.
- Do not pretend a metaphor is a proof.
- Do not oversimplify graduate-level ideas into children's explanations.
- Make the concept usable, not just memorable.
- Keep notation minimal and define each symbol when used.
- State at least one boundary, common mistake, or condition where the concept should not be overextended.

## Output Format

```text
机制故事
{field}

{title}

为什么需要它
{Explain the problem this concept solves.}

小场景
{A concrete situation that makes the need visible.}

机制解释
{Step-by-step explanation. Use clear language. Include a minimal formula if needed.}

小例子
{A short checkable example.}

概念名称
{Concept name}

常见误解或边界
{What this concept does not mean.}

反思问题
{A question that helps the reader apply the mechanism.}
```
